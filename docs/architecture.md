# 🏗️ wk-agent-ops Architecture Guide

> **Project Core Mission**: `wk-agent-ops` is a centralized repository for individuals or teams to develop, manage, and maintain AI Agent enhancements (including Skills, Rules, Workflows, and Git Hooks). The output of this project aims to seamlessly deploy these standardized Agent configurations to other development projects via installation scripts, ensuring consistent, efficient, and highly automated Agent behavior across all projects.

*This file defines the structural integrity of this project. AI agents must respect these boundaries.*

## 1. Core Concept: Template Propagation

wk-agent-ops is a **harness distribution system**. The single source of truth lives in `template/`, and propagates outward via `install.sh` to both this repo's own tooling (`.claude/`) and downstream projects.

```
template/common/        ← Single source of truth
      ↓
scripts/skills/install.sh
      ↓  ↓
.claude/ .agent/        ← This repo's installed tooling (auto-generated)
      ↓
<target-project>/.claude/  ← Other projects (via --target)
```

**Rule:** Modify `template/`, never `.claude/` or `.agent/` directly.

## 2. Directory Structure & Responsibilities

```
wk-agent-ops/
├── template/                  ← Source of truth (edit here)
│   ├── common/                ← Language-agnostic, all projects get this
│   │   ├── .claude/rules/     ← Rules auto-loaded by Claude Code each session
│   │   ├── .github/instructions/  ← Same rules for Copilot CLI
│   │   ├── skills/            ← Shared skills (opsx-commit, doc-updater, etc.)
│   │   ├── AGENTS.md          ← Harness protocol template (no-overwrite on install)
│   │   └── docs/              ← architecture.md, conventions.md starters (no-overwrite)
│   ├── python/                ← Python-specific: pytest hook, style guide
│   └── node/                  ← Node-specific: npm test hook
│
├── scripts/
│   ├── skills/install.sh      ← Propagates template/ to .claude/ and target projects
│   ├── workflow/              ← wt-work, wt-done, wt-resume, pm-start (global CLI tools)
│   │   ├── openspec-branch-creator/  ← PostToolUse hook: auto-creates feature branches
│   │   └── entropy-counter/          ← PostToolUse hook: archive-count threshold banner
│   └── notify/telegram/       ← Telegram notification hook setup
│
├── openspec/
│   ├── specs/                 ← Canonical requirements (source of truth for behaviour)
│   └── changes/               ← Active and archived feature changes
│       └── archive/           ← Completed changes (date-prefixed)
│
├── .claude/                   ← Auto-generated by install.sh (do not edit)
├── .agent/                    ← Auto-generated by install.sh (do not edit)
└── .github/                   ← Auto-generated by install.sh (do not edit)
```

## 3. Two-Phase Workflow Separation

This project enforces a strict **PM / RD separation** via git branches and worktrees:

| Phase | Role | Branch | Tools |
|-------|------|--------|-------|
| PM (Planning) | Write specs, design, tasks | `main` | `pm-start`, `/opsx:new`, `/opsx:ff` |
| RD (Implementation) | Implement tasks | `feature/<name>` worktree | `wt-work`, `/opsx:apply` |

**Why:** Prevents spec artifacts and code from being written in parallel in the same worktree, which causes merge conflicts and context pollution.

---

## 4. Dependency Rules

- **template/ → install.sh → .claude/** — one direction only; no reverse
- **common/ rules apply to all profiles** — python/node profiles extend, never override common
- **Hook scripts call into notify lib** — `scripts/notify/lib/` contains shared config and registry functions; hooks source these, they do not duplicate logic
- **Hooks must be tool-agnostic** — detect `CLAUDE_PROJECT_DIR` vs `GEMINI_PROJECT_DIR` vs `PWD` fallback

---

## 5. Verification Tools

| Tool | Purpose |
|------|---------|
| `bash scripts/skills/install.sh` | Verify template propagation works (use a temp git repo) |
| `/opsx:verify` (Claude Code) | Verify change implementation against specs |
| `/openspec-verify-change` (all tools) | Same, multi-tool compatible |
| `git diff template/common/... .claude/...` | Confirm installed files match source |
